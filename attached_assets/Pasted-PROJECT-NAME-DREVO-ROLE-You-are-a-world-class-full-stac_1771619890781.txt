CORE IDEA (BUSINESS CONTEXT)
Problem:
- Fashion brands carry excess inventory / dead-stock that sits for months and becomes margin-destroying when discounted.
- Consumers can’t find what they want using standard filters; they search by intention (“black oversized tee with a red graphic for a night out”).
Solution:
- A vertical marketplace where brands list products (especially excess inventory) and ship themselves.
- DREVO’s main feature is AI intent search: a chat-like search bar that understands natural language and returns ranked products + simple outfit suggestions.
Business model (for later): marketplace commission 20–30%, Brand Pro subscription (analytics + boosted placement), sponsored placements. Payments can be stubbed in MVP; focus on search + product discovery.

VECTOR SEARCH (CRITICAL)
- Enable pgvector.
- Create ivfflat index for embeddings.
- Create SQL RPC function:
  match_products(query_embedding vector(1536), match_count int, min_similarity float)
returns (product_id uuid, similarity float)
that:
  - searches product_embeddings
  - joins products where products.status='active'
  - returns cosine similarity sorted best-first
- Ensure the function works with Supabase RPC calls.

SECURITY (RLS) — MVP SAFE DEFAULT

  - categories
  - products where status='active'

AI SEARCH PIPELINE (MUST IMPLEMENT)
Endpoint: POST /api/search
Input: { query: string, limit?: number }
Steps:
1) Validate input (non-empty query, limit 1..50).
2) Intent extraction:
   - Call OpenAI chat model to extract a strict JSON object with fields:
     desired_item_type (string optional),
     colors (array of strings),
     style (array),
     fit (string optional),
     occasion (string optional),
     budget_min (number optional),
     budget_max (number optional),
     gender (men|women|unisex optional),
     must_have_keywords (array),
     exclude_keywords (array)
   - MUST return valid JSON, no markdown.
   - Parse JSON safely. If invalid, fallback to empty intent.
3) Embedding creation:
   - Call OpenAI embeddings model with the query string.
4) Vector search:
   - Call Supabase RPC match_products with the embedding vector.
5) Fetch details:
   - Fetch products by IDs, include:
     brand (name, slug),
     images,
     variants,
     tags
6) Ranking:
   - Sort by similarity
   - Optionally boost in-stock
7) Response payload:
{
  query: string,
  intent: object,
  results: [
    {
      id, title, description,
      base_price, sale_price, currency,
      brand: { name, slug },
      images: [{url, position}],
      variants: [{size_label, stock_qty}],
      tags: [string],
      similarity: number,
      reasons: [string]   (explain why it matched, max 3)
    }
  ],
  suggested_filters: { sizes: string[], brands: string[] },
  outfit_bundles: [
    { title: string, items: [productSummary...] }
  ]
}
8) Logging:
   - Insert into search_queries (user_id nullable), store query_text and parsed_intent.
